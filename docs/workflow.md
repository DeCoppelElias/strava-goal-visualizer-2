# TASK EXECUTION PROTOCOL

You MUST follow this sequence strictly:

---

## 1. PLAN MODE
- Understand the task
- Identify missing information
- Ask questions if needed
- Propose approach

🚨 STOP if any uncertainty exists

---

## 2. EXEC MODE
Only after approval:

- Implement minimal solution
- No scope expansion
- No refactors outside task

---

## 3. VALIDATION MODE
- Explain changes
- Provide test steps
- List edge cases

---

## RULES
- One task = one commit
- No mixing tasks
- No hidden refactors
