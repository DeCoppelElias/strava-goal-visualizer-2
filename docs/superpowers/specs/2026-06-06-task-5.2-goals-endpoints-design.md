# TASK-5.2 — GET /goals and PUT /goals Design

_Date: 2026-06-06_

---

## Overview

Add the `goals/` domain with two endpoints that let an authenticated user read and update their yearly running goal. The domain will be extended in TASK-5.3 with progress computation; this task establishes the structure.

## Schemas (`backend/goals/schemas.py`)

```python
class GoalResponse(BaseModel):
    yearly_running_goal_km: float

class UpdateGoalRequest(BaseModel):
    yearly_running_goal_km: float = Field(gt=0, le=100_000)
```

Field name matches the ORM column `Goal.yearly_running_goal_km`. Pydantic returns `422` automatically for values outside `(0, 100_000]`.

---

## Service (`backend/goals/goals_service.py`)

```python
class GoalService:
    async def get_goal(self, db: AsyncSession, user_id: int) -> Goal: ...
    async def update_goal(self, db: AsyncSession, user_id: int, km: float) -> Goal: ...
```

- `get_goal`: `select(Goal).where(Goal.user_id == user_id)` — scalar_one_or_none; raises 404 if missing (defensive only; row is guaranteed by TASK-5.1 auto-creation).
- `update_goal`: fetches goal row, sets `goal.yearly_running_goal_km = km`, returns updated object. No explicit `db.commit()` — transaction ownership stays with the request boundary per the design convention.

---

## Router (`backend/goals/router.py`)

```
GET  /goals  → GoalResponse       (requires get_current_user)
PUT  /goals  → GoalResponse       (requires get_current_user, body: UpdateGoalRequest)
```

Both endpoints use `get_current_user` from `backend.auth.dependencies` — the authenticated user's ID is the only goal that can be read or modified. There is no user_id parameter; a user cannot target another user's goal.

---

## Dependency Injection

`backend/dependencies.py` gets a stateless factory:

```python
def get_goal_service() -> GoalService:
    return GoalService()
```

`GoalService` has no constructor dependencies (no crypto, no OAuth service), so no singleton is needed.

---

## Tests (`tests/backend/goals/test_goals_router.py`)

Uses the same dependency-override pattern as `test_sync_router.py`:

| Test | Assertion |
|---|---|
| GET unauthenticated | 401 |
| GET authenticated, default goal | 200, `yearly_running_goal_km == 365.0` |
| PUT valid value | 200, response reflects new value |
| PUT value ≤ 0 | 422 |
| PUT value > 100,000 | 422 |

Service is mocked via `app.dependency_overrides[get_goal_service]` — no DB required in router tests.

---

## Constraints

- No rate limiting on goals endpoints (backlog does not specify any; auth endpoints and sync are rate-limited but goals are not).
- No `403` case is reachable: `get_current_user` scopes every request to the session user; there is no user_id path or query parameter on these endpoints.
- `Goal` row existence is guaranteed by TASK-5.1 (auto-created on first login). The 404 guard in `get_goal` is defensive only.
