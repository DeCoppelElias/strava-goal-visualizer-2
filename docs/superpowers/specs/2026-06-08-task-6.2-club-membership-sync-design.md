# TASK-6.2 — Club Membership Sync Design

_Date: 2026-06-08_

## Overview

During each `POST /sync`, fetch the authenticated user's Strava clubs and refresh their club membership rows using a replace-all strategy. This ensures membership data is always accurate after every sync.

---

## Strava API

**Endpoint:** `GET https://www.strava.com/api/v3/athlete/clubs`

**Params:** `per_page=200` (single page; no user realistically belongs to 200+ clubs)

**Response shape (relevant fields):**
```json
[
  { "id": 12345, "name": "My Running Club" },
  ...
]
```

**Error handling** follows the same pattern as `fetch_activities`:
- HTTP 401 → `StravaUnauthorizedError`
- Other HTTP errors → `StravaAPIError`
- Network/connection errors (`httpx.RequestError`) → `StravaAPIError`

---

## Architecture

### Option chosen: extend `SyncService` (Option A)

Club sync is a sub-step of the sync operation, not a separate feature. Adding it as a private method on `SyncService` matches the existing `_upsert_activities` / `_upsert_sync_state` pattern and keeps the whole sync atomic in one transaction.

`get_db` wraps the session in `session.begin()`, so clubs, activities, and sync state all commit together or roll back together.

### `backend/sync/strava_client.py`

New function:

```python
async def fetch_athlete_clubs(access_token: str) -> list[dict[str, Any]]:
    # per_page=200 covers any realistic user; Strava's max per page is 200
    ...
```

- Single `GET /athlete/clubs` call with `per_page=200`
- Returns raw list of club dicts
- Error handling identical to `fetch_activities`

### `backend/sync/sync_service.py`

**`run_sync()` — one new line after `_upsert_activities`:**

```python
await self._sync_clubs(db, user_id, access_token)
```

**New private method `_sync_clubs(db, user_id, access_token)`:**

1. Call `fetch_athlete_clubs(access_token)` → raw clubs list
2. If clubs returned: bulk-upsert `clubs` rows via `INSERT ... ON CONFLICT DO UPDATE` on `id`, updating `name` and `updated_at`
3. Delete all existing `club_memberships` for this `user_id` (replace-all — ensures accuracy after every sync)
4. If clubs returned: bulk-insert fresh `club_memberships` rows with `synced_at = now()`

**`SyncResponse` is unchanged** — club count is not exposed to the API consumer.

## Data Flow

```
POST /sync
  └── SyncService.run_sync()
        ├── _check_cooldown()
        ├── ensure_fresh_token()          → access_token
        ├── fetch_all_activities()        → upsert activities
        ├── _sync_clubs()                 ← NEW
        │     ├── fetch_athlete_clubs()   → raw clubs
        │     ├── upsert clubs rows
        │     ├── delete club_memberships for user
        │     └── insert fresh club_memberships
        └── _upsert_sync_state()
```

---

## Membership Replace-All Strategy

On every sync:
1. **Delete** all `club_memberships` rows where `user_id = current_user`
2. **Insert** the fresh set from Strava

This guarantees that clubs a user has left are removed immediately on their next sync. The cost is one extra `DELETE` per sync — negligible.
