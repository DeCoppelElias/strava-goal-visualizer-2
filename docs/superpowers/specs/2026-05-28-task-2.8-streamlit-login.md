# TASK-2.8: Streamlit Login Page and Session State

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the Streamlit auth UI — login page for unauthenticated users, authenticated home page for logged-in users, using the FastAPI session cookie.

**Architecture:** HTTP cookies are domain-based, not port-based. A session cookie set by `localhost:8000` (FastAPI) is sent by the browser to all requests to `localhost`, including the WebSocket upgrade to Streamlit on `localhost:8501`. `st.context.headers.get("cookie")` (Streamlit 1.37+) exposes these headers so Streamlit Python code can forward the cookie in httpx calls to FastAPI. Browser-level redirects (OAuth start, post-logout reload) require JavaScript injection via `components.html`, since Streamlit runs server-side.

**Tech Stack:** Streamlit ≥1.37, httpx, Python 3.12, FastAPI backend (all endpoints already built in TASK-2.4 through TASK-2.7)

```python
import httpx
import streamlit as st
import streamlit.components.v1 as components

from frontend.config import settings

_GDPR_LINKS = "[Privacy Policy](#) · [Terms of Service](#) · [Data Deletion Info](#)"


def _cookie_header() -> dict[str, str]:
    # Cookies are domain-based (not port-based), so the browser sends the
    # FastAPI session cookie (set on localhost:8000) to the Streamlit WebSocket
    # on localhost:8501.  st.context.headers exposes those WebSocket upgrade
    # headers, letting us forward the cookie in server-side httpx calls.
    try:
        cookie = st.context.headers.get("cookie", "")
        return {"cookie": cookie} if cookie else {}
    except AttributeError:
        # st.context requires Streamlit >=1.37; degrade gracefully.
        return {}


def _get_current_user() -> dict | None:
    try:
        response = httpx.get(
            f"{settings.api_base_url}/session/me",
            headers=_cookie_header(),
            timeout=5,
        )
        if response.status_code == 200:
            return response.json()  # type: ignore[no-any-return]
    except httpx.HTTPError:
        pass
    return None


def _render_gdpr_footer() -> None:
    st.markdown("---")
    st.markdown(_GDPR_LINKS)


def _render_login_page(error: str | None) -> None:
    st.title("Strava Goal Visualizer")

    if error == "auth_failed":
        st.error("Authentication failed — please try again.")
    elif error == "strava_error":
        st.error("Strava returned an error — please try again.")

    st.write("Connect your Strava account to visualize your yearly running goal.")

    if st.button("Connect with Strava", type="primary"):
        try:
            resp = httpx.post(f"{settings.api_base_url}/oauth/authorize", timeout=5)
            if resp.status_code == 200:
                url = resp.json()["authorization_url"]
                components.html(
                    f'<script>window.parent.location.href = "{url}";</script>',
                    height=0,
                )
        except httpx.HTTPError:
            st.error("Backend unavailable — please try again.")

    _render_gdpr_footer()


def _render_authenticated_page(user: dict) -> None:
    st.title("Strava Goal Visualizer")
    st.success(f"Connected as Strava athlete #{user['strava_athlete_id']}")

    if st.button("Logout"):
        try:
            httpx.post(
                f"{settings.api_base_url}/session/logout",
                headers=_cookie_header(),
                timeout=5,
            )
        except httpx.HTTPError:
            pass
        # Force a full browser reload so the WebSocket reconnects with the
        # cleared cookie (st.rerun() reuses cached headers, which would still
        # carry the old — now-invalidated — session cookie).
        components.html('<script>window.parent.location.reload();</script>', height=0)

    _render_gdpr_footer()


def main() -> None:
    error = st.query_params.get("error")
    if error:
        st.query_params.clear()

    user = _get_current_user()
    if user:
        _render_authenticated_page(user)
    else:
        _render_login_page(error)


main()
```

## Design Notes

**Why `components.html` for redirects instead of `st.link_button`?**
`st.link_button` would require calling `/oauth/authorize` on every page load (wasting a state token per unauthenticated visit). `st.button` + JS redirect only calls the endpoint when the user actually clicks.

**Why force a page reload after logout instead of `st.rerun()`?**
`st.context.headers` is cached from the WebSocket upgrade — `st.rerun()` reuses those headers. After logout, the browser receives a cleared `Set-Cookie` from FastAPI, but the cached headers still carry the old cookie. Forcing a page reload opens a new WebSocket with the updated (cleared) cookie.

**Why does cookie forwarding work across ports?**
HTTP cookies are matched on domain, not port (RFC 6265). A cookie set by `localhost:8000` without an explicit `Domain` attribute is sent by the browser to any port on `localhost`, including the Streamlit WebSocket on `localhost:8501`.
