# Design: TASK-6.3 — GET /clubs Endpoint

_Date: 2026-06-09_

## Overview

Implement the `GET /clubs` endpoint that returns a list of Strava clubs the authenticated user belongs to. This creates the `backend/clubs/` domain from scratch, following the established domain pattern (schemas, service, router).

## Scope

**In scope:**
- `backend/clubs/` domain: `schemas.py`, `clubs_service.py`, `router.py`, `__init__.py`
- Factory function in `backend/dependencies.py`
- Router registration in `backend/main.py`
- Router tests (mock-based) and service integration tests (real DB via `db` fixture)

**Out of scope:**
- `GET /clubs/{club_id}/progress` (TASK-6.4)
- Any frontend changes (TASK-6.5)

## Data Model

`Club` and `ClubMembership` ORM models already exist in `backend/shared/models.py`. The relevant columns:

- `clubs`: `id` (BigInteger PK, Strava club ID), `name` (text), `updated_at`
- `club_memberships`: `user_id` (FK → users), `club_id` (FK → clubs), `synced_at`; composite PK `(user_id, club_id)`

Membership rows are populated during `POST /sync` (TASK-6.2). No membership rows → endpoint returns `[]`.

## Architecture

### `backend/clubs/schemas.py`

```python
class ClubResponse(BaseModel):
    id: int
    name: str
```

### `backend/clubs/clubs_service.py`

`ClubsService` with one method:

```python
async def get_clubs(self, db: AsyncSession, user_id: int) -> list[Club]:
    result = await db.execute(
        select(Club)
        .join(ClubMembership, ClubMembership.club_id == Club.id)
        .where(ClubMembership.user_id == user_id)
    )
    return list(result.scalars().all())
```

Single JOIN query. Returns `[]` naturally when the user has no memberships. Does not commit — caller owns the session.

### `backend/clubs/router.py`

```python
@router.get("/clubs", response_model=list[ClubResponse])
@limiter.limit("30/minute")
async def get_clubs(
    request: Request,
    current_user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
    clubs_service: ClubsService = Depends(get_clubs_service),
) -> list[ClubResponse]:
    clubs = await clubs_service.get_clubs(db, current_user.id)
    return [ClubResponse(id=c.id, name=c.name) for c in clubs]
```

### `backend/dependencies.py`

```python
def get_clubs_service() -> ClubsService:
    return ClubsService()
```

### `backend/main.py`

Import and register `clubs_router` alongside the existing routers.

## Query Design

Chosen approach: **JOIN query** (Option A). One DB round-trip regardless of club count. Matches the pattern used by `GoalService` and `DashboardService`. The relationship traversal alternative would re-fetch the `User` row and require `selectinload`, adding overhead with no benefit.

## Error Handling

- No sync / no memberships → returns `200 []` (not a 404; the user is valid, they just have no clubs)
- Unauthenticated → `401` via `get_current_user`
- Rate limit exceeded → `429` via slowapi

## Testing

**`tests/backend/clubs/test_clubs_router.py`** — mock-based (no DB):
- Unauthenticated request returns `401`
- Authenticated request with clubs returns `200` with correct JSON shape
- Authenticated request with no clubs returns `200 []`

**`tests/backend/clubs/test_clubs_service.py`** — integration tests using `db` fixture:
- Seed `User` + `Club` + `ClubMembership` rows → `get_clubs` returns those clubs
- User with no memberships → returns `[]`
- User A's clubs are not returned for User B (isolation)

Integration tests are required per project convention: `get_clubs` emits SQL, so assertions must be on real DB row state.
