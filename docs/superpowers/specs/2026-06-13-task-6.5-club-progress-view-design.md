## `ClubsPage` Component

**Props:** None. Clubs are session-scoped ‚Äî `GET /clubs` requires no parameter beyond the session cookie.

**State:**

| State | Type | Purpose |
|---|---|---|
| `clubs` | `Club[]` | List of user's clubs, fetched on mount |
| `selectedClubId` | `number \| null` | Auto-set to first club after fetch |
| `clubDashboard` | `ClubDashboard \| null` | Fetched when `selectedClubId` changes |
| `clubsStatus` | `'loading' \| 'error' \| 'loaded'` | Fetch status for clubs list |
| `dashStatus` | `'loading' \| 'error' \| 'loaded'` | Fetch status for selected club dashboard |

**Data flow:**
1. On mount: fetch `GET /clubs` ‚Üí populate `clubs`, auto-select first, set `clubsStatus`
2. When `selectedClubId` changes (via dropdown or auto-select): fetch `GET /dashboard/club/{id}` ‚Üí set `clubDashboard`, set `dashStatus`
3. Dropdown `onChange` ‚Üí update `selectedClubId` ‚Üí triggers dashboard fetch

**Layout (top to bottom):**

1. **Page header** ‚Äî `<h1 className="page-title">Clubs</h1>` + mono subtitle (club name or empty)

2. **Club selector card** ‚Äî `card` with `card__label` "Your Clubs":
   - `<select className="club-select">` populated from `clubs`
   - Empty state (no clubs): `"No clubs found ‚Äî sync your data first."`
   - Loading state: disabled select showing "Loading‚Ä¶"

3. **Disclaimer banner** ‚Äî always visible when a club is selected, rendered as:
   ```
   status-msg status-msg--warning
   "This club view shows members who have connected this app.
    It is a progress visualization, not a competition leaderboard."
   ```

4. **Members card** ‚Äî `card` with `card__label` "Members":
   - Loading state: `dash-loading` text "Loading‚Ä¶"
   - Error state: `status-msg--danger` + Retry button
   - Empty state (0 members): `"No other members of this club have connected the app yet."`
   - Member rows (see below)

5. **No chart in this task** ‚Äî placeholder comment noting TASK-6.6 will add it here.

**Member row structure:**
```
[Display name]  [‚ñà‚ñà‚ñà‚ñà‚ñà‚ñà‚ñà‚ñà‚ñë‚ñë‚ñë‚ñë‚ñë‚ñë]  [42% ¬∑ 153 / 365 km]
```
- Name: `14px`, `--text-1`, `min-width: 120px`
- Bar track: `flex: 1`, `6px` tall, `--surface-2` bg, pill radius
- Bar fill: `--accent`, width = `clamp(0%, progress_pct, 100%)`, `300ms ease` transition
- Stats: JetBrains Mono `12px`, `--text-2`, `white-space: nowrap`
  - Format: `{progress_pct.toFixed(1)}% ¬∑ {distance_to_date_km.toFixed(1)} / {goal_km.toFixed(0)} km`

## Error Handling

| Scenario | Behaviour |
|---|---|
| `GET /clubs` fails | `clubsStatus = 'error'`, error card with Retry |
| `GET /clubs` returns `[]` | Empty state: "No clubs found ‚Äî sync your data first." |
| `GET /dashboard/club/{id}` fails | `dashStatus = 'error'`, error card with Retry inside members card |
| `members` array is empty | Empty state: "No other members of this club have connected the app yet." |
