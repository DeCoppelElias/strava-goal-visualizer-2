# TASK-7.3 — Privacy Export Endpoint Design

_Date: 2026-06-13_

## Overview

Add `POST /privacy/export` — an authenticated endpoint that assembles all stored data for the requesting user and returns it as a downloadable JSON file. No new tables. Rate limited to 5/hour to prevent scraping.

## Architecture

Three changes:

- `backend/privacy/privacy_service.py` — add `export_user_data(db, user_id) -> dict`
- `backend/privacy/router.py` — new file; thin route that calls the service, serializes with `json.dumps`, returns `Response`
- `backend/main.py` — include privacy router

No `schemas.py` — the endpoint returns a raw `Response`, not a Pydantic model. Serialization of `Decimal` and `datetime` is handled in the router via a custom `json.dumps` `default` function.

## Payload Structure

```json
{
  "exported_at": "2026-06-13T10:00:00+00:00",
  "user": {
    "strava_athlete_id": 12345678,
    "display_name": "Elias D.",
    "created_at": "2026-01-01T00:00:00+00:00"
  },
  "goal": {
    "yearly_running_goal_km": 500.0
  },
  "sync_state": {
    "last_sync_completed_at": "2026-06-13T09:00:00+00:00"
  },
  "activities": [
    {
      "strava_activity_id": 123456,
      "name": "Morning Run",
      "sport_type": "Run",
      "distance_meters": 5000.0,
      "moving_time_seconds": 1800,
      "start_date": "2026-06-13T07:00:00+00:00"
    }
  ],
  "club_memberships": [
    {
      "club_id": 42,
      "synced_at": "2026-06-13T09:00:00+00:00"
    }
  ]
}
```

**Excluded fields:** Internal auto-increment PKs (`id`) and foreign keys (`user_id`) are never included. OAuth tokens are never included.

**Null/empty handling:**
- `goal` → `null` if no goal row exists
- `sync_state` → `null` if no sync has occurred
- `activities` → `[]` if none
- `club_memberships` → `[]` if none

## Router

```python
# backend/privacy/router.py
@router.post("/privacy/export")
@limiter.limit("5/hour")
async def export_user_data(request, current_user, db, privacy_service) -> Response:
    data = await privacy_service.export_user_data(db, current_user.id)
    payload = json.dumps(data, default=_serialize, indent=2)
    return Response(
        content=payload,
        media_type="application/json",
        headers={"Content-Disposition": 'attachment; filename="strava-export.json"'},
    )

def _serialize(obj):
    if isinstance(obj, Decimal): return float(obj)
    if isinstance(obj, datetime): return obj.isoformat()
    raise TypeError(f"Object of type {type(obj)} is not JSON serializable")
```

## Error Handling

`get_current_user` covers the `401` case. No other failure modes — missing optional rows return `null` or `[]`, never raise.

## Testing

Integration tests using the `db` fixture:

1. **Happy path** — seed user with goal, activities, club membership, sync state → call `export_user_data` → assert all fields present with correct values; `id` and `user_id` absent; no token fields.
2. **Sparse user** — user with no goal, no sync state, no activities, no memberships → `goal: null`, `sync_state: null`, `activities: []`, `club_memberships: []`.
