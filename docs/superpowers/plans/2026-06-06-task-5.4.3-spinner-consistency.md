# TASK-5.4.3 — Spinner Consistency: Rename Class and Add Logout Spinner

**Goal:** Two UX inconsistencies to fix:
1. `.sync-btn__spinner` is reused in the goal-save button — rename to `.btn__spinner` (generic, correct scope).
2. The logout button shows plain `"…"` while in-flight; all other async buttons show a spinner — add one.

---

## File Map

| File | Action |
|------|--------|
| `frontend/src/index.css` | Modify — rename CSS class |
| `frontend/src/pages/DashboardPage.tsx` | Modify — update two class references |
| `frontend/src/pages/HomePage.tsx` | Modify — add spinner to logout button |

---

### Step 1 — Rename CSS class in `index.css`

```css
/* Before */
.sync-btn__spinner { ... }

/* After */
.btn__spinner { ... }
```

---

### Step 2 — Update class references in `DashboardPage.tsx`

Two usages to update (sync button and save button):

```tsx
// Before (both occurrences)
<span className="sync-btn__spinner" aria-hidden="true" />

// After (both occurrences)
<span className="btn__spinner" aria-hidden="true" />
```

---

### Step 3 — Add spinner to logout button in `HomePage.tsx`

```tsx
// Before
<button className="btn btn--ghost" onClick={handleLogout} disabled={loggingOut}>
  {loggingOut ? '…' : 'Log out'}
</button>

// After
<button className="btn btn--ghost" onClick={handleLogout} disabled={loggingOut}>
  {loggingOut ? (
    <>
      <span className="btn__spinner" aria-hidden="true" />
      Logging out…
    </>
  ) : (
    'Log out'
  )}
</button>
```

Note: The ghost button spinner inherits `border-top-color: #fff`. In light mode the ghost button has dark text so the spinner will be less visible. Add a variant to `index.css` for ghost buttons if this looks wrong during verification — but check first, as the spinner border is semi-transparent and may look fine against either background.
