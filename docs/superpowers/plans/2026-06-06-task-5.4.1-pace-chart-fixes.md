# TASK-5.4.1 — Fix PaceChart Light-Mode Fill and Leap-Year Domain

**Goal:** Fix two bugs in `PaceChart.tsx`: the area fill is hardcoded to the dark-mode colour so it doesn't adapt to light mode, and the X-axis domain hardcodes `365` (wrong in leap years).

**Architecture:** `buildChartData` already computes `daysInYear` — expose it in the return value so `PaceChart` can pass it to `<XAxis domain>`. Read `--accent-dim` from `getComputedStyle` (same pattern as `--accent`) and use it as `<Area fill>`.

---

## File Map

| File | Action |
|------|--------|
| `frontend/src/components/PaceChart.tsx` | Modify — expose `daysInYear`, read `--accent-dim`, fix `fill` and `domain` |

### Step 1 — Return `daysInYear` from `buildChartData`

Change the return type from `ChartPoint[]` to `{ data: ChartPoint[]; daysInYear: number }`:

```tsx
function buildChartData(
  dailySeries: DailyDistancePoint[],
  goalKm: number,
): { data: ChartPoint[]; daysInYear: number } {
  const year = new Date().getFullYear()
  const isLeap = (year % 4 === 0 && year % 100 !== 0) || year % 400 === 0
  const daysInYear = isLeap ? 366 : 365

  // ... rest of function body unchanged ...

  return {
    data: [...byDay.values()].sort((a, b) => a.day - b.day),
    daysInYear,
  }
}
```

### Step 2 — Read `--accent-dim` in the component

In `PaceChart`, add `accentDim` next to the existing computed-style reads:

```tsx
const accentDim = style.getPropertyValue('--accent-dim').trim() || 'rgba(75,140,247,0.08)'
```

Destructure the new return shape:

```tsx
const { data, daysInYear } = buildChartData(dailySeries, goalKm)
```
