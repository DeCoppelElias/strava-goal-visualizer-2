# Tasks 3.3.1–3.3.6 + get_db auto-commit: Auth/Sync Layer Cleanup — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Harden the auth/sync layer with timezone safety, error handling, exception consolidation, pagination, and correct transaction ownership before the sync endpoint (TASK-3.4) is built.

**Architecture:** Seven sequential tasks. 3.3.4 creates `shared/exceptions.py` first (other tasks import from it). 3.3.1–3.3.5 are additive. 3.3.6 removes `db.commit()` from `ensure_fresh_token`. Task 7 moves all transaction management into `get_db` via `session.begin()`, removing every remaining explicit `db.commit()` call. Each task = one commit.

**Tech Stack:** Python 3.12, FastAPI, SQLAlchemy async ORM, httpx, pytest (`asyncio_mode = "auto"`), respx 0.21+, Alembic

> **Key discovery (3.3.1):** Migration `0001_create_auth_tables.py` already creates `token_expires_at` and `expires_at` as `DateTime(timezone=True)`. The DB columns are already TIMESTAMPTZ. **No new Alembic migration is needed** — the fix is ORM model-only.

```python
async def fetch_all_activities(
    access_token: str,
    *,
    after: int | None = None,
) -> list[dict[str, Any]]:
    all_activities: list[dict[str, Any]] = []
    page = 1
    per_page = 200
    while True:
        page_results = await fetch_activities(
            access_token,
            after=after,
            page=page,
            per_page=per_page,
        )
        all_activities.extend(page_results)
        if len(page_results) < per_page:
            break
        page += 1
    return all_activities
```
