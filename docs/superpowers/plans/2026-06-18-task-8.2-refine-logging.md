# TASK-8.2 — Refine Application Logging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make production logs traceable to a single request (request-id correlation) and emit explicit, actionable records at the real failure points (Strava 429, OAuth token refresh, deauth webhook), keeping plain-text output.

**Architecture:** A `contextvar` holds a per-request id; a `logging.Filter` injects it onto every record so the format string can print it; a Starlette middleware sets the contextvar (and echoes an `X-Request-ID` header) for the duration of each request. Logging setup moves out of `main.py` into `backend/shared/logging.py`. Failure-point log statements are added at three existing sites. No tokens are ever logged.

**Tech Stack:** Python `logging` (stdlib), `contextvars`, Starlette `BaseHTTPMiddleware`, FastAPI; tests use `pytest`, `respx`, `caplog`, `TestClient`.

**Spec:** `docs/superpowers/specs/2026-06-18-observability-basics-design.md`

**One-commit rule:** Per `CLAUDE.md`, one backlog task = one commit. Build and verify all tasks, then make a **single commit** in the final task. Do not commit between Tasks 1–5.

**Test note:** All tests in this plan use mocks / `respx` / `TestClient` and do **not** use the `db` fixture, so they run without the Postgres testcontainer. The final full-suite run (Task 6) does need Docker (already running).

```python
import logging
from contextvars import ContextVar

request_id_var: ContextVar[str] = ContextVar("request_id", default="-")

LOG_FORMAT = "%(asctime)s %(levelname)s [%(request_id)s] %(name)s — %(message)s"


class RequestIdFilter(logging.Filter):
    """Inject the current request id onto every log record so LOG_FORMAT can print it."""

    def filter(self, record: logging.LogRecord) -> bool:
        setattr(record, "request_id", request_id_var.get())
        return True


def configure_logging() -> None:
    """Configure root logging with the request-id-aware plain-text format."""
    logging.basicConfig(level=logging.INFO, format=LOG_FORMAT)
    request_id_filter = RequestIdFilter()
    for handler in logging.getLogger().handlers:
        handler.addFilter(request_id_filter)
```

Note: `setattr` (not `record.request_id = ...`) keeps mypy strict happy, since `LogRecord` has no declared `request_id` attribute.

```python
import uuid

from starlette.middleware.base import BaseHTTPMiddleware, RequestResponseEndpoint
from starlette.requests import Request
from starlette.responses import Response

from backend.shared.logging import request_id_var

REQUEST_ID_HEADER = "X-Request-ID"


class RequestIdMiddleware(BaseHTTPMiddleware):
    """Assign a request id (incoming header or generated), expose it via the
    contextvar for logging, and echo it back on the response."""

    async def dispatch(self, request: Request, call_next: RequestResponseEndpoint) -> Response:
        request_id = request.headers.get(REQUEST_ID_HEADER) or uuid.uuid4().hex
        token = request_id_var.set(request_id)
        try:
            response = await call_next(request)
        finally:
            request_id_var.reset(token)
        response.headers[REQUEST_ID_HEADER] = request_id
        return response
```
